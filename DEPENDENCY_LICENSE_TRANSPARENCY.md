# Dependency License Transparency

Review date: 2026-05-29

This document maps the repository's observed direct dependencies by scope. It
is based on `pyproject.toml`, `scripts/build_macos_app.py`, README media
references, and repository files present at review time.

This is not a full SBOM. It does not enumerate transitive Python packages,
system frameworks, macOS platform libraries, optional Ollama models, or
packages installed in a developer's local environment.

## Scope and Assumptions

- The project uses `pyproject.toml` as the dependency source of truth.
- There is no lockfile in the repository, so resolved versions can drift.
- Dependencies declared with lower bounds can resolve to newer package
  versions with changed metadata.
- Optional extras and local services are not active runtime dependencies unless
  a user installs/enables them.
- The lightweight macOS `.app` launcher depends on the Python environment it
  points to; its redistributed dependency obligations depend on what is
  bundled with that environment.

## Known Compliance Gaps

| Gap | Impact | Recommended remediation |
| --- | --- | --- |
| No lockfile | Resolved versions and license metadata can drift across installs. | Generate and retain a release lockfile or resolved dependency report. |
| No transitive dependency license report | Direct dependency mapping is not enough for release due diligence. | Generate `pip-licenses`, CycloneDX, or equivalent reports for runtime, optional, and test environments. |
| PySide6/Qt packaging obligations are not captured as release evidence | Redistributed `.app` bundles may miss LGPL notice, source, relinking, or replacement obligations. | Add a release checklist for Qt/PySide notices and bundle inspection. |
| Optional `docling` dependency is unpinned | Optional installs can resolve to changing versions and transitive dependencies. | Pin or record the resolved `docling` version in release evidence when the optional backend is shipped or documented as supported. |
| Ollama model licenses are not mapped | Local model terms may restrict redistribution, commercial use, or generated-output handling. | Document supported model names and review their licenses separately if examples or defaults are added. |
| Media asset provenance is implicit | Future reuse or redistribution may lack proof of authorship. | Add a short media provenance note or asset inventory if GIFs are externally sourced or regenerated. |

## Suggested Release-Time Verification Commands

Run these commands in the exact environment intended for packaging:

```bash
python -m pip install pip-licenses
python -m piplicenses --format=markdown --with-urls --with-system
python -m pip freeze
python -m pip show PySide6-Essentials pypdf openpyxl platformdirs pytest ruff
python -m pip show docling
```

When building a packaged macOS app, inspect the resulting bundle and release
environment for bundled Qt/PySide libraries, Python packages, license files,
and generated notices before publishing.
